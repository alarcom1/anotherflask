CI/CD

Project

For my project I used bootstrap v5 for css. I have a yml file that pushes my project image to heroku twice. One for Dev and one for Prod as a design.

GitHub Actions

In my website project under the ".github/workflows" folder I have a dev.yml and a prod.yml that is synchronized with a heroku deploy API that I have setup in my Github repository. In Github settings > secrets > actions.

In my flask website application I use the "CI/CD" process using GitHub actions that when my master branch is updated, pushed, or pull requested, it triggers a workflow to update the heroku prod and dev app

In PyCharm services terminal, I run a "pytest" that runs through a series of checks I have specified in "simple_pages_test.py" python file.

Badges for production workflow and development workflow are displayed in my github repository specified on the "Readme.MD" file.

Code Review

In GitHub Actions Workflows, I display a series of workflows that run through test checks specified in my .yml files that runtests, build containers, and deploys. In summary, the tests;

  • set up job
  • run actions/checkout
  • set up python 3.8
  • install dependencies
  • test with pytest
  • post run actions/checkout@v2
  • complete job

I am able to deploy an image of my project to Dockerhub when my master branch is updated to GitHub because of the prod.yml file with the build container, runs on, and steps. Configurations where I specified the dockerhub image, login, and deploy commands.
